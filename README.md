# 인스타툰 홀덤

인스타툰 작가들끼리 하는 온라인 노리밋 홀덤. 참가할 때 인스타그램 아이디를 입력하면
게시물 수에 비례해서 시작 칩을 받습니다.

- 시작 칩 = **1000 + 게시물 수 × 50** (최대 +5000까지)
- 인스타 데이터는 로그인 없이 프로필 공개 페이지의 메타태그(og:description, og:image)만 읽어옵니다.
  비공개 계정은 조회되지 않습니다.
- 2~9명, 사이드팟 포함 정식 텍사스 홀덤 규칙.

## 로컬 실행

```bash
npm install
npm start
```

브라우저에서 `http://localhost:3000` 접속. 방장이 "방 만들기"로 코드를 발급하고,
나머지는 그 코드로 "방 참가하기" 하면 됩니다. 같은 와이파이/네트워크가 아니어도
아래 배포를 하면 누구나 접속 가능합니다.

## 칩 배분 공식 바꾸기

`server.js` 상단의 `CHIP_RULE` 값을 수정하면 됩니다.

```js
const CHIP_RULE = { base: 1000, perPost: 50, cap: 5000 };
```

## 무료로 배포해서 친구들과 같이 하기

Render.com 기준 (무료 플랜 가능):

1. GitHub에 이 폴더를 올린 새 저장소를 만듭니다.
2. https://render.com 가입 → New → Web Service → 방금 만든 저장소 선택
3. Build Command: `npm install`, Start Command: `npm start`
4. 배포되면 나온 URL을 친구들에게 공유하면 끝.

(Railway, Fly.io, Glitch 등 Node.js를 지원하는 다른 무료/저가 호스팅도 동일하게 동작합니다.)

## 알아두면 좋은 점

- 인스타그램 공개 페이지를 자동으로 읽어오는 방식이라, 인스타그램이 페이지 구조를 바꾸면
  일시적으로 프로필 조회가 안 될 수 있습니다. 이 경우 "프로필 미리보기"에서 에러 메시지가 뜹니다.
- 짧은 시간에 너무 많은 사람이 동시에 조회하면 인스타그램이 일시적으로 요청을 막을 수 있어요.
  소규모 친구 모임용으로는 문제없는 수준입니다.
- 서버는 방 상태를 메모리에만 저장합니다. 서버를 재시작하면 진행 중이던 방은 초기화됩니다.
- 재접속/새로고침해도 같은 브라우저 탭이 유지되는 동안은 그대로 게임에 남아있지만,
  탭을 닫으면 자동으로 폴드 처리되고 자리에서 빠집니다(재입장 기능은 미포함).

## 파일 구조

```
server.js              Express + Socket.io 서버, 방 관리
game/pokerEngine.js     홀덤 규칙 엔진 (베팅, 사이드팟, 쇼다운)
game/instagram.js       인스타그램 프로필 파싱 + 시작 칩 계산
public/index.html       클라이언트 화면
public/app.js           클라이언트 로직 (Socket.io 연동, 렌더링)
public/style.css        스타일
```
